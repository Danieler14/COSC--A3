Group Reflections

Cassie

What went well

I think what really went well this time in comparison to the last assignment was splitting up the group. Since its such a large group of 7 people it had made it extremely difficult to organise meetings where everyone could attend. Splitting the group up made it not only easier to organise smaller group meetings but also spread out the workload better so that 1 group could focus on the app development and the other could focus on the website development.

What could be improved

I think we underestimated how difficult it would have been to get 4 people to work on the website at once, and we did end up changing it so that there were 2 main people actually designing the prototype, and then 2 others supporting with ideas and suggestions and working more on the document.

One thing that was surprising

One thing I found surprising was the level of detail needed in order to be able to create a functioning app and website login page. I went into this assignment thinking we may be able to use github with HTML and CSS to produce the PTV Rewards app, but I was extremely wrong. The UI and skills needed to create something like this is far more advanced than I would have expected.

One thing I learnt about groups

One thing I have learned about groups is that when you pull back and don't micromanage as much, others will step up to the plate. I have learned that I need to trust my group members more and relax a bit.

Github Log of Activity

Most of our GitHub log activity was done towards the end. The way we have structured how this works is we make sure all components are done, reviewed and good to go and then it's uploaded. All parts uploaded should be either a part someone worked on or a part that they did.

Andrew

What went well

Organisation was a lot better when compared to our last assignment. Right from the start, everyone was quick to be vocal about what they wanted to take part in making it immensely easier to assign certain portions of the project.

What could be improved

An improvement that seems to be impossible to fix was trying to find adequate time for meetings to take place. It was hard to find a good time to have every member at every meeting but we managed to think of a partial solution which involved dividing the group into two (for the website and app) and having meetings in those assigned groups.

One thing that was surprising

A thing I found surprising was how extensive creating an app and website is. The amount of roles and work that is required for these developments is insane, considering a lot of it is done behind the scenes. It made me see both app and web developments through a new lens, having a new appreciation for even the more negligible aspects I used to look over.

One thing I learnt about groups

One thing I learned about group work would be that subdividing a big group into smaller groups can increase efficiency and communication tremendously. I found it more engaging to have small group discussions as everyone was given more time to talk and work out solutions to problems we encountered.

Github Log of Activity

Most of, if not all, our GitHub log of activity took place towards the end of the project. Proof-reading and adjustments were made using Google Docs and then sent to the GitHub repository. Overall, GitHub was a great way to incorporate every aspect of the project as everyone played a part in adding their part to the repository.
